# Thinking Types: System Overview
## For internal alignment and teacher-facing design

---

## Part 1: Data Sources — Where Do the Signals Come From?

We don't invent new assessments. We extract thinking signals from **data Amplify already collects**.

### Source A: mCLASS / DIBELS 8 (3x per year)

| Assessment | How It's Collected | What It Captures |
|-----------|-------------------|-----------------|
| **ORF** (Oral Reading Fluency) | Teacher listens to student read aloud for 1 min on iPad; taps correct/incorrect per word | Reading speed (WPM), accuracy (%), per-word error transcript, self-correction behavior, pace variability |
| **Maze** (Comprehension) | Student silently reads a passage for 3 min; picks the correct word from 3 choices at blanks | Adjusted score, literal vs inferential item accuracy, response time patterns |

**Key signals we extract beyond standard scores:**
- ORF pace variability → steady reader vs variable reader (Structure dimension)
- ORF self-correction rate → metacognition signal (Depth dimension)
- Maze inferential vs literal accuracy → surface comprehension vs deep comprehension (Depth dimension)

### Source B: Amplify ELA Writing (Continuous — multiple essays per unit)

| Data Point | How It's Collected | What It Captures |
|-----------|-------------------|-----------------|
| **AWE Scores** (Automated Writing Evaluation) | Student submits essay through platform; AI scores Focus, Evidence, Elaboration, Conventions, Language | Quality of argument construction, evidence use, reasoning depth |
| **Essay Text** | Stored in platform | Content analysis: connective words, cross-text references, argument structure, vocabulary diversity |
| **Revision History** | Platform tracks drafts | Surface edits (spelling/grammar) vs structural revision vs argument revision |
| **Cross-Essay Patterns** | Compared across units | Idea evolution over time, compositional range, consistency vs experimentation |

**Key signals we extract:**
- AWE Evidence score + how evidence is used → convergent (few, deep) vs divergent (many, broad)
- Essay structure consistency across units → systematic vs exploratory
- Connective word diversity ("First, Second, In conclusion" vs varied transitions) → structure dimension
- Cross-text connections in writing → depth of processing
- Revision type patterns → surface edits = systematic; restructuring = exploratory

## Part 2: Three Dimensions — The "Axes" of Thinking

Like MBTI has 4 axes (E/I, S/N, T/F, J/P), we have **3 dimensions**:

```
         Dimension 1              Dimension 2              Dimension 3
    ┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
    │  DEPTH           │     │  STRUCTURE       │     │  EVIDENCE        │
    │                  │     │                  │     │                  │
    │  Surface ← → Deep│     │  Systematic ← → │     │  Convergent ← → │
    │                  │     │    Exploratory   │     │    Divergent     │
    │  "How deeply do  │     │  "How do they    │     │  "How do they   │
    │   they engage?"  │     │   organize?"     │     │   use evidence?"│
    └─────────────────┘     └─────────────────┘     └─────────────────┘
```

| Dimension | Low End | High End | What It Measures |
|-----------|---------|----------|-----------------|
| **Depth** | Surface — restates, lists facts | Deep — connects, transfers, creates meaning | How deeply the student processes information |
| **Structure** | Systematic — planned, consistent, follows conventions | Exploratory — emergent, flexible, unconventional | How the student organizes their thinking |
| **Evidence** | Convergent — one strong position, decisive | Divergent — multiple angles, comfortable with ambiguity | How the student handles evidence and perspectives |

## Part 3.1: The Cube — How 8 Types Relate to Each Other

The 8 types are not arbitrary labels. They form a **3D cube** with three kinds of meaningful relationships:

```
                         DEEP
                      ┌─────────────────────────────────┐
                     /│ 🔥 Advocate          🧭 Explorer │
                    / │ (Expl+Conv)          (Expl+Div)  │
                   /  │                                  │
                  /   │ 🏛️ Architect         🔍 Detective│
                 /    │ (Syst+Conv)          (Syst+Div)  │
                /     └─────────────────────────────────┘
         SURFACE     /                                 /
         ┌─────────────────────────────────┐          /
         │ ⚡ Improviser        🌊 Wanderer │        /
         │ (Expl+Conv)          (Expl+Div)  │       /
         │                                  │      /
         │ 📋 Reporter          📚 Collector│     /
         │ (Syst+Conv)          (Syst+Div)  │    /
         └─────────────────────────────────┘
```

### Relationship 1: Neighbors (differ by 1 dimension)

Types that share 2 of 3 dimensions are **neighbors** — connected by an edge of the cube. Each type has exactly 3 neighbors.

```
  🏛️ Architect ←→ 🔍 Detective     (differ: Evidence)
  🏛️ Architect ←→ 🔥 Advocate      (differ: Structure)
  🏛️ Architect ←→ 📋 Reporter      (differ: Depth)
  🔍 Detective ←→ 🧭 Explorer      (differ: Structure)
  🔍 Detective ←→ 📚 Collector     (differ: Depth)
  🔥 Advocate  ←→ 🧭 Explorer      (differ: Evidence)
  🔥 Advocate  ←→ ⚡ Improviser    (differ: Depth)
  🧭 Explorer  ←→ 🌊 Wanderer      (differ: Depth)
  📋 Reporter  ←→ 📚 Collector     (differ: Evidence)
  📋 Reporter  ←→ ⚡ Improviser    (differ: Structure)
  📚 Collector ←→ 🌊 Wanderer      (differ: Structure)
  ⚡ Improviser ←→ 🌊 Wanderer     (differ: Evidence)
```

**Every edge is a possible growth direction.** Which edge matters depends on what this student needs right now:

| Move | What It Looks Like | When a Teacher Might Want This |
|------|-------------------|-------------------------------|
| Reporter → Architect | Add depth to their organized facts | Student is reliable but never analyzes |
| Reporter → Collector | Broaden their evidence base | Student is stuck on one source |
| Reporter → Improviser | Loosen up, take creative risks | Student is rigid and afraid to experiment |
| Architect → Detective | Open up to multiple perspectives | Student is rigorous but one-sided |
| Explorer → Detective | Add structure to their creativity | Student has great ideas but can't organize them |
| Wanderer → Explorer | Go deeper on what interests them | Student touches many ideas but stays shallow |
| Wanderer → Improviser | Learn to commit to a position | Student can't make a choice |
| Detective → Advocate | Learn to commit after analyzing | Student over-researches and never concludes |

**The teacher decides the growth direction, not the system.** The system shows where the student is; the teacher knows where they need to go.

### Relationship 2: Opposites (differ on all 3 dimensions)

| Type | Opposite | Why They Complement |
|------|----------|-------------------|
| 🏛️ Architect | 🌊 Wanderer | Maximum rigor meets maximum openness |
| 🔍 Detective | ⚡ Improviser | Careful analysis meets quick instinct |
| 🔥 Advocate | 📚 Collector | Deep conviction meets broad collection |
| 🧭 Explorer | 📋 Reporter | Creative connections meets factual precision |

**Why this matters:** Opposites are the **best peer-review pairs**. Each has exactly what the other needs.

### How This Differs from MBTI

| | MBTI | Thinking Types |
|---|---|---|
| Dimensions | 4 (all stylistic) | 3 (all stylistic — Depth describes preference, not ability) |
| Types | 16 (fixed for life) | 8 (dynamic — changes across tasks, units, and years) |
| Growth | No defined direction — an INTJ doesn't "grow into" another type | **Every edge of the cube is a growth direction.** The teacher chooses based on the student's needs. |
| Judgment | Some types are culturally perceived as "better" | Explicitly non-hierarchical. A Reporter's precision is as valuable as an Explorer's creativity. |

## Part 4: Character Profiles — The 8 Thinkers

### Design Concept
Like MBTI characters, each type gets a **unique color palette**, **character persona**, and **visual identity** that teachers (and eventually students) can recognize instantly. No type is "better" — each has strengths and growth areas.

**Color: Navy Blue**
**Character: A young builder with blueprints and a compass**

| | |
|---|---|
| Dimensions | Deep × Systematic × Convergent |

| Strength | Rigor, clarity, persuasion |
| Growth area | Taking creative risks, flexibility |

| | |
|---|---|

**Color: Sky Blue / Light Blue**
**Character: A young journalist with a notepad, writing down facts**

| | |
|---|---|

| Strength | Reliability, organization, factual accuracy |
| Growth area | Moving beyond "what happened" to "why it matters" |

| | |
|---|---|

## Part 5: Color System at a Glance

Each type gets its own distinct color — **no hierarchy implied by saturation or warmth**.

```
  🏛️ Architect    ███  Navy Blue         📋 Reporter     ███  Sky Blue
  🔍 Detective    ███  Emerald Green      📚 Collector    ███  Amber Gold
  🔥 Advocate     ███  Coral Red          ⚡ Improviser   ███  Orange
  🧭 Explorer     ███  Teal              🌊 Wanderer     ███  Lavender
```

**Color logic:**
- **8 distinct, equally vibrant colors** — no type looks "faded" or "lesser"
- **Neighbors share a warm/cool tone** to hint at proximity on the cube, but each type stands on its own
- **Opposites have maximum color contrast** (Navy ↔ Lavender, Emerald ↔ Orange, Coral ↔ Amber, Teal ↔ Sky Blue) — reinforcing the complementary relationship

## Part 6: How Types Are Detected

### Step 1: Score each dimension (0-10)

Each dimension score is computed from observable signals. Every signal traces back to **specific raw data fields** that Amplify already collects.

---

#### Dimension 1: Depth of Processing

| Signal | Weight | Raw Data Source | How to Compute | Difficulty |
|--------|--------|----------------|---------------|-----------|
| Maze inferential accuracy | 0.15 | `maze.responses[]` — per-item correctness | Each Maze item is pre-tagged as literal or inferential in item metadata. Compute: inferential_correct / inferential_total. | Easy — requires item tagging (one-time), then simple ratio |
| AWE Evidence score | 0.20 | `awe_scores.evidence` — already scored 1-4 | Normalize to 0-10 scale. Direct read from existing system. | Easy — data already exists |
| Reasoning in writing | 0.25 | `essay.text` — full essay content | Detect causal reasoning chains: "because", "therefore", "this shows that", "as a result". Count reasoning sentences / total sentences. | Medium — keyword matching gets you 70%; LLM gets you 95% |
| Cross-text connections | 0.20 | `essay.text` + `unit.reading_list` | Detect references to texts outside the current assignment. Compare essay content against reading list from other units. | Hard — needs NLP/LLM to detect paraphrased references, not just exact quotes |
| Idea evolution | 0.20 | `essay.text` across multiple units for same student | Compare stance/argument on similar topics across essays. Are ideas developing or copy-pasted? | Hard — requires cross-essay semantic comparison, LLM recommended |

```
depth_score = (
    maze_inferential_accuracy     × 0.15 +
    awe_evidence_score            × 0.20 +
    reasoning_in_writing          × 0.25 +
    cross_text_connections        × 0.20 +
    idea_evolution                × 0.20
) × 10
```

---

#### Dimension 2: Structure (Systematic ↔ Exploratory)

| Signal | Weight | Raw Data Source | How to Compute | Difficulty |
|--------|--------|----------------|---------------|-----------|
| Structural consistency | 0.25 | `essay.text` across multiple essays | Represent each essay as a shape vector: [paragraph_count, avg_sentences_per_paragraph, intro_length, body_length, conclusion_length]. Cosine similarity across essays. High similarity = systematic. | Medium — structural parsing is straightforward; comparing across essays needs pipeline |
| Connective diversity | 0.20 | `essay.text` | Build a transition word dictionary ("First", "However", "In conclusion", etc.). Count unique connectives / total connectives. Low diversity = systematic. | Easy — keyword dictionary + ratio |
| Revision type | 0.25 | `essay.revision_history[]` — diff between drafts | Classify each edit: spelling/grammar fix (surface), sentence reorder (structural), argument change (deep). Ratio of structural+deep edits / total edits. High ratio = exploratory. | Medium — diff is easy; classifying edit *type* needs NLP |
| Paragraph organization | 0.15 | `essay.text` | Detect whether each paragraph has a clear topic sentence (first sentence states the paragraph's point). Score: paragraphs with topic sentences / total paragraphs. High = systematic. | Medium — heuristic works OK; LLM is better |
| ORF pace variability | 0.15 | `orf.measure_transcript[]` — per-word timing (if available) | Standard deviation of reading pace across the passage. Proxy: error_rate_first_30s vs error_rate_last_30s. High variability = exploratory reader. | Easy if timing data exists; otherwise use error distribution as proxy |

```
structure_score = (
    (1 - structural_consistency)  × 0.25 +    // invert: high consistency = low score = systematic
    connective_diversity          × 0.20 +
    deep_revision_ratio           × 0.25 +
    (1 - paragraph_organization)  × 0.15 +    // invert: good organization = low score = systematic
    orf_pace_variability          × 0.15
) × 10
// low score = systematic, high score = exploratory
```

---

#### Dimension 3: Evidence Orientation (Convergent ↔ Divergent)

| Signal | Weight | Raw Data Source | How to Compute | Difficulty |
|--------|--------|----------------|---------------|-----------|
| Number of evidence sources | 0.25 | `essay.text` | Count distinct quotes or text references. More sources = more divergent. | Medium — quote detection (quotation marks, "according to", "the text says") |
| Counterargument present | 0.25 | `essay.text` | Detect language acknowledging other viewpoints: "some might argue", "on the other hand", "however, others believe". Binary (0 or 1) or graded (0-1 based on depth). | Medium — keyword matching + context check |
| Position consistency | 0.20 | `essay.text` across essays on related topics | Compare stance on similar themes across units. Consistent position = convergent. Shifting position = divergent. | Hard — needs LLM to understand stance, not just surface words |
| Maze answer changes | 0.15 | `maze.responses[]` — need answer change logging | Count times student selected an answer, then changed it. More changes = more divergent (weighing options). | Easy if platform logs changes; may not exist today |
| Depth per evidence | 0.15 | `essay.text` | For each piece of evidence cited, count sentences of explanation. Few sources + many explanation sentences = convergent. Many sources + few explanation sentences = divergent. | Medium — needs evidence detection + surrounding sentence counting |

```
evidence_score = (
    normalize(num_evidence_sources) × 0.25 +
    counterargument_present         × 0.25 +
    (1 - position_consistency)      × 0.20 +    // invert: consistent = convergent = low
    maze_answer_changes             × 0.15 +
    (1 - depth_per_evidence)        × 0.15      // invert: deep per source = convergent = low
) × 10
// low score = convergent, high score = divergent
```

#### Feasibility Summary

| Difficulty | Signals | What's Needed |
|-----------|---------|--------------|
| **Easy** (5 signals) | AWE Evidence score, connective diversity, ORF pace variability, Maze answer changes, Maze inferential accuracy | Direct data read or simple keyword matching. Can ship in Phase 1. |
| **Medium** (6 signals) | Reasoning in writing, structural consistency, revision type, paragraph organization, num evidence sources, counterargument present, depth per evidence | NLP rules + keyword patterns get you most of the way. Improve with LLM later. Phase 1-2. |
| **Hard** (3 signals) | Cross-text connections, idea evolution, position consistency | Require cross-document semantic analysis. LLM-based. Phase 2-3. |

#### Phase 1 Approach (ship with what we have)

In Phase 1, use only Easy + Medium signals. For the 3 Hard signals, substitute simpler proxies:

| Hard Signal | Phase 1 Proxy | Tradeoff |
|------------|--------------|---------|
| Cross-text connections | Count explicit quotes from non-assigned texts (exact string match) | Misses paraphrased references, but catches obvious ones |
| Idea evolution | Cosine similarity of essay embeddings across units — high similarity = static, low = evolving | Crude but directionally correct |
| Position consistency | Compare AWE Focus scores + claim sentences across essays — same claim structure = consistent | Misses nuanced stance shifts |

### Step 2: Map scores to type (with boundary zones)

Each dimension score maps to one of three zones:

```
  0 ──── 3.9          4.0 ──── 6.0          6.1 ──── 10
  ◄─── Clear pole A ──►◄── Boundary zone ──►◄── Clear pole B ──►
```

| Dimension | 0 – 3.9 | 4.0 – 6.0 | 6.1 – 10 |
|-----------|---------|-----------|---------|
| Depth | Surface | Boundary | Deep |
| Structure | Systematic | Boundary | Exploratory |
| Evidence | Convergent | Boundary | Divergent |

### Step 3: Determine classification result

The number of dimensions in the boundary zone determines what the student sees:

| Boundary dimensions | Result | Example |
|--------------------|--------|---------|
| **0** — all clear | **Primary type** — one of 8 types, high confidence | "🏛️ Architect" |
| **1** — one ambiguous | **Primary + secondary type** — the two types on either side of the boundary dimension | "Most like 🧭 Explorer, also shows 🔍 Detective traits" |
| **2-3** — mostly ambiguous | **No type assigned** — show the three dimension scores directly | "Profile still forming — Depth: 7.2, Structure: 4.8, Evidence: 5.1" |

### Confidence score

Each classification also carries a confidence value (0-1) based on:

```
confidence = f(
    signal_count,                    // more writing samples = more confident
    distance_from_boundaries,        // scores far from 4.0-6.0 = more confident
    consistency_across_recent_tasks  // same type across tasks = more confident
)
```

### Minimum data required

- 3+ writing samples in current unit
- 1+ mCLASS data point in current year
- If not enough → shows "Not enough data yet" instead of any profile

## Part 7: Recommendations — Where Types Become Action

The framework is only useful if it changes what teachers **do**. Three concrete scenarios:

---

### Scenario 1: Smart Grouping

**When:** Teacher is setting up a collaborative writing task or peer review for the next unit.

**Where in the product:** Class dashboard → "Suggested Groups" tab (new surface)

**What the teacher sees:**

```
┌──────────────────────────────────────────────────────────┐
│  Unit 6D Writing Task: Peer Review Partners              │
│                                                          │
│  SUGGESTED PAIRS (based on complementary strengths)      │
│                                                          │
│  🏛️ James (Architect)  ←→  🌊 Chloe (Wanderer)          │
│  "James can help Chloe organize her ideas.               │
│   Chloe can help James see new angles."                  │
│                                                          │
│  🔍 Elena (Detective)  ←→  🔥 Zara (Advocate)            │
│  "Elena can push Zara to consider other viewpoints.      │
│   Zara can help Elena commit to a position."             │
│                                                          │
│  📋 Tyler (Reporter)   ←→  🧭 Priya (Explorer)           │
│  "Tyler grounds discussions in facts.                    │
│   Priya models how to connect facts to bigger ideas."    │
│                                                          │
│  [Edit Groups]  [Accept & Assign]                        │
└──────────────────────────────────────────────────────────┘
```

**Logic:**
- Default pairing = **opposites on the cube** (differ on all 3 dimensions) — maximum complementarity
- Teacher can override any pairing — the system suggests, never dictates
- Each pair gets a one-sentence explanation of *why* — so the teacher can judge if it makes sense for these particular students

**Alternative grouping modes:**
| Mode | Logic | When to Use |
|------|-------|------------|
| Complementary pairs | Pair opposites | Peer review, collaborative writing |
| Similar clusters | Group same types | When students need to feel "I'm not the only one who thinks this way" |
| Stretch groups | Pair neighbors (1 dimension apart) | Gentle push — student sees a style close to theirs but slightly different |

---

### Scenario 2: Type-Aware Feedback

**When:** Student submits an essay and receives AWE feedback.

**Where in the product:** Student's writing feedback screen (existing surface, enhanced logic)

**What changes:** The feedback is the same interface students already see. But the system **selects different feedback priorities** based on the student's current type — without revealing the type to the student.

**Example — same essay, different feedback depending on type:**

A student writes a solid 3-paragraph essay about Greek democracy with one quote from the text.

| If the student is a... | Feedback emphasis | Example feedback |
|------------------------|------------------|-----------------|
| 📋 Reporter | Push toward analysis | "You summarized the key facts clearly. Now try adding a sentence about *why* this matters — what does Greek democracy tell us about today?" |
| ⚡ Improviser | Push toward evidence | "You have a strong opinion here. Can you find a quote from the text that supports your point?" |
| 🏛️ Architect | Push toward flexibility | "Your argument is well-structured. What would someone who *disagrees* with you say? Try adding one sentence that considers another view." |
| 🌊 Wanderer | Push toward focus | "You raised several interesting ideas. Which one matters most to you? Try building your next draft around just that one idea." |
| 🔍 Detective | Push toward commitment | "You've examined this from many angles — great analysis. Now: what's *your* conclusion? Take a stand." |
| 🔥 Advocate | Push toward nuance | "Your passion comes through clearly. To make your argument even stronger, try acknowledging one reason someone might see it differently." |
| 📚 Collector | Push toward synthesis | "You found great sources. Now try connecting them: how does Source A relate to Source B? What do they tell us *together*?" |
| 🧭 Explorer | Push toward structure | "You made a creative connection between Greek democracy and modern voting. Now try organizing your ideas: what's your main point, and what evidence supports it?" |

**Key constraints:**
- Student never sees their type name
- Feedback language is always positive and specific ("try adding..." not "you failed to...")
- One growth nudge per submission — don't overwhelm
- Teacher can preview and edit the feedback before it's sent to the student

### Scenario 3: Drift Detection (Early Warning)

**When:** After each unit's writing is complete, the system updates each student's type and checks for significant shifts.

**Where in the product:** Teacher dashboard → "Alerts" or class overview (new surface)

**What the teacher sees:**

```
┌──────────────────────────────────────────────────────────┐
│  ⚠️  TYPE SHIFTS — Mr. Thompson's Class, Unit 6D         │
│                                                          │
│  NOTABLE CHANGES (3 students)                            │
│                                                          │
│  Maria Chen                                              │
│  6A: 🧭 Explorer → 6B: 🔍 Detective → 6C: 🧭 Explorer    │
│  → 6D: 📋 Reporter                                       │
│  ▲ Shifted from Deep to Surface on informational writing.│
│    May need support engaging analytically with            │
│    unfamiliar content.                                    │
│                                                          │
│  Jake Williams                                           │
│  6A: ⚡ Improviser → 6B: ⚡ Improviser → 6C: ⚡ Improviser│
│  → 6D: ⚡ Improviser                                     │
│  ▲ Stable for 4 units. Strong voice, but hasn't          │
│    developed evidence use. Consider targeted              │
│    scaffolding for citing text evidence.                  │
│                                                          │
│  Sofia Park                                              │
│  6A: 📋 Reporter → 6B: 📚 Collector → 6C: 🔍 Detective   │
│  → 6D: 🏛️ Architect                                      │
│  ✓ Consistent growth across all 3 dimensions.            │
│    Current approach is working well.                      │
│                                                          │
│  [View Full Class]  [Dismiss]                            │
└──────────────────────────────────────────────────────────┘
```

**Alert logic — what triggers a flag:**

| Pattern | Alert Type | What It Means |
|---------|-----------|--------------|
| Deep → Surface shift | ⚠️ Warning | Possible disengagement, content difficulty, or external factors |
| Same type 3+ units in a row | ℹ️ Info | Stable preference — may benefit from a stretch activity |
| Movement across multiple dimensions | ✓ Positive | Student is developing range and flexibility |
| Sudden type change not explained by task type | ⚠️ Warning | Worth a conversation with the student |

**What the alert is NOT:**
- Not a grade or evaluation
- Not a mandate ("you must intervene")
- It's a **conversation starter** — "I noticed your writing changed in this unit. What's going on?"

**Key constraint:** The system flags patterns. The teacher interprets them. A shift from Explorer to Reporter might mean disengagement — or it might mean the student is learning to be precise with unfamiliar content. Only the teacher knows the full context.


The three scenarios cover the full task cycle:
- **Before** the writing task → group students strategically
- **During** the task → give type-aware feedback
- **After** the task → detect shifts and flag concerns

## Key Design Principles

1. **Types, not scores** — "You're an Explorer" not "Inferential Reasoning: 4/10"
2. **Dynamic, not fixed** — A student can be an Architect in one unit and an Explorer in the next
3. **No type is better** — Every type has genuine strengths. A Reporter's precision is as valuable as an Explorer's creativity. The language is always asset-based.
4. **Growth is multi-directional** — There is no single "best" type to grow toward. Every edge of the cube is a valid growth direction. The teacher decides what this student needs next.
5. **Students never see the label** — Only teachers see types. Students get tailored feedback and scaffolding.
6. **Teachers make the calls** — The system shows where a student is. The teacher decides where they should go. Pedagogical judgment stays with the teacher.
